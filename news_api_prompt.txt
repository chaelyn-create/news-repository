1. 코드 생성을 위한 프롬프트 
------------------------------------------------------------------------------

너는 파이썬(Python), Streamlit, Supabase, Gemini API를 능숙하게 다루는 풀스택 AI 개발자이자 친절한 코딩 선생님이야. 로컬(내 컴퓨터)에 파이썬을 설치하지 않고, 웹 브라우저만으로 GitHub와 Streamlit Cloud를 이용해 "AI 최신 뉴스 검색 & 자동 저장기" 프로젝트를 바로 배포하려고 해. 아래 4가지 항목을 완벽하게 작성해서 순서대로 출력해 줘.

---[요청 항목 1: Supabase DB 생성 SQL]
- Supabase의 SQL Editor에 바로 붙여넣을 수 있는 `CREATE TABLE` 구문을 작성해 줘.
- 테이블명: `news_history`
- 컬럼: id(기본키, 자동증가), keyword, title, source, news_date, url, summary, created_at(기본값 now)
- 중요: url 컬럼은 중복 저장을 막기 위해 반드시 `UNIQUE` 제약 조건을 걸어줘.

---[요청 항목 2: requirements.txt 파일]
- 이 앱을 실행하기 위해 필요한 파이썬 라이브러리 목록을 작성해 줘. 
- 필수 포함: streamlit, pandas, google-genai, supabase

---[요청 항목 3: app.py 전체 파이썬 코드]
아래 요구사항을 완벽히 반영한 하나의 전체 코드를 작성해.
1. 기술 스택: Streamlit, Supabase, Google GenAI SDK (`google-genai` 라이브러리의 `genai.Client`)
2. 현재 Gemini API에서는 'Google 검색 도구(Tool)'와 '강제 JSON 응답 모드(response_mime_type)'를 동시에 켜는 것을 지원하지 않는 점을 유의해줘. 
3. 시크릿 관리: `st.secrets`에서 "GEMINI_API_KEY", "SUPABASE_URL", "SUPABASE_KEY"를 불러와 사용.
4. 화면 구성: 3개의 탭 (Tab 1: 🔍 검색하기, Tab 2: 💾 저장된 뉴스 보기, Tab 3: 📊 통계 분석)
5. Tab 1 요구사항 (검색 및 저장)
   - 키워드 입력 후 버튼을 누르면 Gemini API (`gemini-2.5-flash`)를 호출. 도구는 `google_search` 활성화, 환각 방지를 위해 Temperature는 0.0.
   - 프롬프트 지시: "키워드에 대한 가장 최신 뉴스 딱 **2건**만 검색해. 제목, 출처, 날짜, 원본 URL, 요약을 포함한 JSON 배열로 응답하고 절대 URL을 지어내지 마."
   -[URL 환각 완벽 방지 로직 (매우 중요)]: `response.candidates[0].grounding_metadata.grounding_chunks`를 순회하여 실제 참조한 기사 제목(`chunk.web.title`)과 실제 링크(`chunk.web.uri`)를 추출하고, 생성된 JSON의 제목과 비교하여 일치하면 JSON의 'url' 값을 실제 링크로 덮어쓰는 파이썬 로직을 반드시 추가해. 단, 구글 내부 임시 링크로 인한 404 접속 에러를 방지하기 위해 grounding-api-redirect를 절대 쓰지 마. 실제 링크(real_url)가 "http"로 시작하고 "grounding-api-redirect"를 포함하지 않을 때만 덮어쓰도록 조건을 걸어줘. 
   - 2건의 뉴스를 화면에 카드형태로 출력.
   - Supabase `news_history` 테이블에 데이터를 insert 하되, URL 중복 에러(23505) 발생 시 에러 대신 "중복 생략됨"으로 카운트. (작업 완료 후 st.toast로 결과 알림)
6. Tab 2 요구사항 (저장된 뉴스 보기)
   - DB에서 데이터를 최신순으로 가져와 Pandas DataFrame으로 변환 후 `st.dataframe`으로 표출.
   - 제목/키워드 기준 검색 필터링 창 추가.
   - 현재 화면의 데이터를 CSV로 다운로드하는 버튼 추가.
7. Tab 3 요구사항 (통계 대시보드)
   - 왼쪽 컬럼: DB 데이터 기반 '키워드별 누적 검색 건수' (`st.bar_chart`)
   - 오른쪽 컬럼: 날짜(YYYY-MM-DD) 추출 후 '일자별 저장 건수' (`st.line_chart`)

---[요청 항목 4: 초보자를 위한 노코드 웹 배포 튜토리얼]
코딩을 모르는 학생도 마우스 클릭만으로 웹에 배포할 수 있도록 다음 전체 프로세스를 단계별로 아주 쉽게 설명해 줘. (로컬 코드 실행 불필요 강조)
1. Gemini API Key 발급받기 (Google AI Studio)
2. Supabase 가입, 프로젝트 생성, SQL Editor로 테이블 만들기, API URL 및 anon key 복사해두기
3. GitHub 가입 및 새 레포지토리(Repository) 생성, 웹상에서 바로 `app.py`와 `requirements.txt` 파일 만들고 코드 붙여넣어 저장(Commit)하기
4. Streamlit Community Cloud(share.streamlit.io) 가입, GitHub 연동 후 'Deploy an app' 하기
5. Advanced settings (또는 Secrets)에 들어가서 아래 형식으로 3개의 환경변수 입력하는 방법 강조
   GEMINI_API_KEY = "내_키"
   SUPABASE_URL = "내_주소"
   SUPABASE_KEY = "내_키"
6. 앱 배포 완료! 이후 코드 수정이 필요하면 내 컴퓨터가 아닌 GitHub 웹사이트에서 직접 `app.py`를 수정(Edit)하고 Commit만 하면 Streamlit이 자동 업데이트된다는 점 설명하기.

3. Supabase SQL -- news_history 테이블 생성
------------------------------------------------------------------------------

CREATE TABLE news_history (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    keyword TEXT NOT NULL,
    title TEXT NOT NULL,
    source TEXT,
    news_date TEXT,
    url TEXT UNIQUE NOT NULL,
    summary TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
